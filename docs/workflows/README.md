## Placement Rule

Keep a document in `workflows/` when it answers one of these questions:

- how to perform a repeated repo/app task
- how a large workflow is currently staged
- where a deep-dive companion belongs for a specific workflow family

If the file is the repo-wide rule for a topic, keep it in `docs/` root instead and link back here only as needed.
