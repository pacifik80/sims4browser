## Required record fields

The first reopened transparent-object fixture should record, at minimum:

1. named object identity
2. reopened root identity
3. companion resources that survived reopen
4. current winning transparent-family branch
5. contradictory losing signals, if any
6. the exact signal that made the winner stronger
7. whether the fixture satisfies the current promotion boundary

## Minimum field checklist

### Identity block

- package name
- object name
- object-side identity or swatch anchor, if available
- reopened root

### Reopen block

- whether reopen was stable
- whether the same path can be followed again restart-safely
- companion bundle that survived reopen

### Classification block

- winning branch:
  - object-glass
  - threshold/cutout
  - `AlphaBlended`
  - or only if truly necessary `SimGlass`
- winning signal
- losing or contradictory signals

### Promotion block

- does the fixture cross the current promotion threshold: yes or no
- if no, which threshold item is still missing
