## Required provisional record

The first provisional `Build/Buy SimGlass` candidate must record, at minimum:

1. stable reopened root
2. restart-safe object identity
3. surviving transparent-object scope evidence
4. the provisional `SimGlass` signals that stayed alive
5. the stronger object-side branches that did not win cleanly
6. the exact missing evidence that still blocks a `SimGlass` win
7. why the result is still narrower than generic transparent fallback

## Minimum field checklist

### Identity block

- package name
- object name
- object-side identity or swatch anchor, if available
- reopened root

### Reopen block

- whether reopen was stable
- whether the path is restart-safe
- surviving companion bundle

### Provisional-signal block

- provisional surviving `SimGlass` signal
- contradictory or competing branch signals
- exact reason those competing branches did not win cleanly

### Blocker block

- which missing evidence still blocks promotion into a winning `SimGlass` fixture
- whether the result would collapse to generic transparent fallback if that surviving `SimGlass` signal disappeared

## Safe reading

- a provisional candidate is not a weakly worded win
- it is a bounded intermediate result with explicit remaining blockers
